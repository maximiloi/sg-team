# SG Team — сайт автосервиса Super Garage Team

Веб-приложение для автосервиса **Super Garage Team** с интеграцией Telegram-бота для уведомлений о новых заявках.

**Сайт:** [sgt-service.ru](https://sgt-service.ru/)

---

## 📋 Описание

Проект представляет собой современную CRM-систему для автосервиса с функционалом:

- **Лендинг** — информация об услугах, ценах, преимуществах и отзывах
- **Онлайн-запись** — форма записи на сервис с выбором типа услуги и автомобиля
- **Личный кабинет** — авторизация и регистрация пользователей
- **Админ-панель** (`/board`) — управление заявками и записями
- **Telegram-бот** — уведомления менеджера о новых заявках
- **История действий** — логирование всех изменений в системе

## 🚀 Запуск проекта

### Требования

- **Node.js** 20+
- **PostgreSQL** 14+
- **Telegram Bot Token** (для бота)

### Установка

```bash
# Клонирование репозитория
git clone <repository-url>
cd sg-team

# Установка зависимостей
npm install

# Генерация Prisma клиента
npm run db:generate

# Применение миграций
npm run db:migrate

# Запуск разработки
npm run dev
```

### Переменные окружения

Создайте файл `.env` в корне проекта (скопируйте из `.env.example`):

```bash
cp .env.example .env
```

Заполните файл `.env`:

```env
# База данных
DATABASE_URL="postgresql://user:password@localhost:5432/db_name"

# NextAuth
NEXTAUTH_URL="http://localhost:3000"
AUTH_SECRET="your-secret-key"
```

> **Примечание:** Для генерации `AUTH_SECRET` используйте команду:
>
> ```bash
> openssl rand -base64 32
> ```
>
> или:
>
> ```bash
> node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
> ```

```env
# Telegram бот
TELEGRAM_BOT_TOKEN="your-bot-token"
TELEGRAM_MANAGER_IDS="123456789,987654321"
```

## 🛡️ Безопасность

### Rate Limiting (защита от спама)

Проект использует встроенную систему rate limiting для защиты от спама и brute force атак:

| Endpoint / Action                      | Лимит                | Блокировка |
| -------------------------------------- | -------------------- | ---------- |
| Форма заявки (`/api/clients`)          | 3 заявки в минуту    | 5 минут    |
| Запись на сервис (`createAppointment`) | 5 записей в минуту   | 5 минут    |
| Регистрация (`signupAction`)           | 3 попытки в 15 минут | 30 минут   |
| Вход в систему                         | 5 попыток в 15 минут | 15 минут   |

**Заголовки ответа при rate limit:**

- `X-RateLimit-Limit` — максимальное количество запросов
- `X-RateLimit-Remaining` — осталось запросов
- `X-RateLimit-Reset` — время сброса счётчика (timestamp)
- `Retry-After` — секунд до повторной попытки

### Валидация данных

Все входные данные валидируются через **Zod схемы**:

- Телефон: формат `+7 (XXX) XXX-XX-XX`
- VIN: 17 символов (без I, O, Q)
- Email: валидный email формат
- Пароль: минимум 8 символов, цифры, заглавные и строчные буквы

### Санитизация

Строковые данные санитизируются перед сохранением в БД (защита от XSS):

- Экранирование HTML-символов (`&`, `<`, `>`, `"`, `'`)
- Trim пробелов

## 🗄 База данных

### Модели данных (Prisma)

| Модель          | Описание                               |
| --------------- | -------------------------------------- |
| **User**        | Пользователи системы (админы, техники) |
| **Client**      | Клиенты автосервиса                    |
| **Request**     | Заявки на обратный звонок              |
| **Car**         | Автомобили клиентов (VIN, госномер)    |
| **Appointment** | Записи на сервис                       |
| **ActionLog**   | История действий пользователей         |

### Роли пользователей (UserRole)

| Роль         | Описание                        |
| ------------ | ------------------------------- |
| `ADMIN`      | Администратор с полным доступом |
| `TECHNICIAN` | Техник/мастер                   |

### Статусы заявок (RequestStatus)

| Статус        | Описание                     |
| ------------- | ---------------------------- |
| `NEW`         | Новая заявка                 |
| `IN_PROGRESS` | В работе                     |
| `CALL_CLIENT` | Требуется звонок клиенту     |
| `DONE`        | Выполнена                    |
| `CALLBACK`    | Не дозвонились (перезвонить) |
| `REJECTION`   | Отказ клиента                |
| `CANCELLED`   | Отменена / спам              |

### Типы записей (AppointmentType)

| Тип           | Описание        |
| ------------- | --------------- |
| `MAINTENANCE` | Техобслуживание |
| `DIAGNOSTICS` | Диагностика     |
| `REPAIR`      | Ремонт          |

### Статусы записей (AppointmentStatus)

| Статус        | Описание      |
| ------------- | ------------- |
| `SCHEDULED`   | Запланирована |
| `CONFIRMED`   | Подтверждена  |
| `IN_PROGRESS` | В работе      |
| `DONE`        | Выполнена     |
| `CANCELLED`   | Отменена      |
